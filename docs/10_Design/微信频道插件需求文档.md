## 一、产品概述

### 1.1 产品定位

微信频道插件是一个让用户通过微信与自己本地部署的 OpenClaw AI 进行对话的桥接工具。

### 1.2 核心功能

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│     微信        │────▶│   中转服务      │────▶│   OpenClaw      │
│  (用户端)       │◀────│  (云端)         │◀────│  (本地AI)       │
└─────────────────┘     └─────────────────┘     └─────────────────┘
        │                       │                       │
   发送消息                WebSocket转发            处理消息
   接收回复                消息路由                 返回AI响应
```

### 1.3 用户场景

1. **用户安装客户端插件** → 本地运行
2. **扫码授权** → 绑定微信
3. **微信发消息** → 本地 AI 处理 → 微信收回复

## 二、安装流程设计

### 2.1 安装方式

支持一键安装命令：

```bash
# macOS/Linux（必须使用 bash）
curl -fsSL https://wechat.clawadmin.org/release/install.sh | bash

# Windows PowerShell
irm https://wechat.clawadmin.org/release/install.ps1 | iex
```

### 2.2 安装步骤

```
Step 1: 检查 Python 环境 (需要 3.8+)
Step 2: 检查 pip
Step 3: 下载客户端模块
        - src/client.py
        - src/watchdog.py
        - src/updater.py
        - src/wechat_types.py
        - src/update_state.py
        - requirements.txt (根目录)
Step 4: 创建虚拟环境
Step 5: 安装依赖 (websockets, httpx)
Step 6: 创建启动脚本
```

### 2.3 下载源优先级

| 优先级 | 源 | 适用场景 |
|--------|-----|---------|
| 1 | Gitee | 国内用户首选 |
| 2 | Cloudflare CDN | 全球加速 |
| 3 | GitHub | 备用 |
| 4 | 国内备用 | 最后备选 |

## 三、技术架构

### 3.1 客户端架构

```
wechat-channel/
├── client.py          # 主客户端逻辑
├── watchdog.py        # 看门狗（监控重启）
├── updater.py         # 自动更新
├── wechat_types.py    # 类型定义（避免与标准库冲突）
├── update_state.py    # 更新状态管理
├── requirements.txt   # 依赖清单
├── venv/              # 虚拟环境
├── start.sh           # 启动脚本
├── stop.sh            # 停止脚本
├── uninstall.sh       # 卸载脚本
├── config.json        # 设备配置
└── logs/              # 日志目录
```

### 3.2 WebSocket 通信协议

**客户端 → 服务端（注册）**:
```json
{
    "type": "register",
    "instance_type": "local",
    "device_id": "bare_xxx_username_timestamp_random",
    "device_type": "bare",
    "machine_id": "硬件指纹",
    "system_username": "用户名",
    "client_version": "版本号",
    "min_server_version": "最低服务端版本",
    "is_new_device": true
}
```

**服务端 → 客户端（响应）**:
```json
{
    "type": "registered",
    "server_version": "x.x.x",
    "is_recovery": false,
    "auth_url": "https://xxx/auth?device=xxx"
}
```

### 3.3 设备 ID 格式

```
{type}_{machine_id}_{system_username}_{timestamp}_{random4}

示例: bare_82fe9c38f255a124_bluth_20260318162206_63c8
```

- **type**: bare | ecs | docker_local | docker_cloud
- **machine_id**: 16位硬件指纹
- **system_username**: 系统用户名
- **timestamp**: YYYYMMDDHHMMSS
- **random4**: 4位随机字符

## 四、配置项

| 环境变量 | 默认值 | 说明 |
|---------|--------|------|
| `OPENCLAW_URL` | `http://127.0.0.1:18789` | OpenClaw 服务地址 |
| `RELAY_URL` | `wss://claw.7color.vip/ws-channel` | 中转服务地址 |
| `INSTANCE_TYPE` | `local` | 实例类型 |

## 五、已知限制

1. **必须使用 bash 执行安装脚本** - 脚本使用 bash 数组语法，不兼容 sh/dash
2. **Python 3.8+** - 使用了较新的 asyncio 特性
3. **需要网络访问** - 连接中转服务和下载源
